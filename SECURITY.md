# Security Policy

## Reporting a Vulnerability

If you believe you have discovered a security vulnerability in this project,
please report it privately. **Do not open a public issue.**

Report via email to **security@uttera.ai** with the following information:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

You can expect:

- Acknowledgement of your report within 72 hours
- A first assessment within 7 days
- Coordinated disclosure once a fix is available

We appreciate responsible disclosure. Contributors of valid reports will be
credited in release notes unless they request anonymity.

## Supported Versions

Only the latest minor release receives security updates. Older versions
may contain unpatched vulnerabilities and should not be used in production.

## Scope

This policy covers the code in this repository. It does NOT cover:

- Third-party dependencies (report those upstream)
- Model weights (report to the model authors)
- Deployments of this code operated by third parties (report to the operator)
